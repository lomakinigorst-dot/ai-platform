# Модуль AI-Маркетолог — как работает при добавлении клиента

## Концепция

Маркетолог — **базовый модуль**, который запускается автоматически при добавлении нового клиента.
Его результаты используют все остальные блоки (консультант, HR, финансы и т.д.) — как фундамент знаний о бизнесе клиента.

---

## Шаги добавления нового клиента (для интегратора)

### Шаг 1 — Добавить сайт
Интегратор вводит URL сайта клиента в форму.
- Можно открыть несколько вкладок параллельно → добавлять несколько клиентов одновременно
- Каждая вкладка = независимый процесс сканирования
- Вкладку закрывать нельзя пока идёт сканирование (используются ресурсы браузера)

### Шаг 2 — Сканирование сайта
После нажатия «Сканировать» запускается краулер:
- Обходит страницы сайта (главная, услуги, о компании, контакты, каталог)
- Собирает тексты, заголовки, мета-теги, структуру страниц
- Прогресс виден: «Просканировано 12 из ~30 страниц»
- Статус: 🔄 Сканирование → ✅ Готово

### Шаг 3 — Автозапуск ДНК-анализа (AI-Маркетолог)
Сразу после окончания сканирования — автоматически запускается AI-анализ по ДНК-промпту.
Без участия интегратора. DeepSeek обрабатывает блоками последовательно:

**Блок 1 — Анализ ниши**
Промпт: «Что за компания, чем занимается, какой продукт/услуга, регион, позиционирование»
→ Результат сохраняется в карточку клиента: вкладка «Ниша»

**Блок 2 — Глубокий анализ ЦА**
ДНК-промпт (5 частей из файла) — обучение AI роли маркетолога-психолога.
Затем вопросы: цели, боли, страхи, триггеры, возражения, мифы, эмоциональные мотивы.
→ Вкладка «ЦА»

**Блок 3 — Аватары клиентов**
На основе анализа ЦА — AI строит 3-5 детальных аватаров.
→ Вкладка «Аватары»

**Блок 4 — Сценарии поиска**
6 частей: сценарии поиска, типология формулировок, семантические кластеры, негативные запросы, сленг/ошибки, поисковые цепочки.
→ Вкладка «Сценарии поиска»

**Блок 5 — Сегментация**
7 сегментов: кто он, где его найти, что важно, какой тон, примеры рекламных подходов.
→ Вкладка «Сегменты»

**Блок 6 — РСЯ / Заголовки / УТП**
По каждому сегменту: 30 заголовков, 30 описаний РСЯ, 30 ключей, 30 визуалов, 30 УТП.
→ Вкладка «Заголовки и УТП»

### Шаг 4 — Автозаполнение настроек AI-консультанта
На основе результатов маркетинг-анализа DeepSeek автоматически формирует:

- **Системный промпт AI-чата** — кто агент, как отвечает, стиль общения, что предлагает
- **Инструкции агента** — на что закрывать, какие боли использовать, какие возражения отрабатывать
- **Приветственное сообщение** — персонализированное под нишу
- **Быстрые кнопки** — топ-3 вопроса посетителей (из сценариев поиска)
- **FAQ** — автогенерация из контента сайта и болей ЦА

Интегратор может отредактировать всё это вручную — но по умолчанию уже заполнено.

### Шаг 5 — WOW-момент: демо-ссылка
После шагов 3 и 4 — автогенерируется демо-страница с рабочим виджетом.
Интегратор отправляет клиенту: «Смотрите, ваш AI-консультант уже настроен»

## Статусы в карточке клиента

| Этап | Статус | Иконка |
|---|---|---|
| Сайт добавлен | Ожидание сканирования | ⏳ |
| Сканирование идёт | Сканирование... N страниц | 🔄 |
| Сканирование готово | Анализ ЦА... | 🔄 |
| ДНК-анализ готов | Настройка AI... | 🔄 |
| Всё готово | Активен | ✅ |
| Ошибка | Требует внимания | ⚠️ |

## Технические детали

- **Краулер:** запускается в браузере интегратора (Playwright/Puppeteer в Web Worker) — не нагружает сервер
- **AI:** DeepSeek API — дёшево, быстро для длинных промптов
- **Очерёдность блоков:** каждый блок ДНК-промпта отправляется только после получения ответа на предыдущий
- **Хранение:** результаты каждого блока сохраняются по мере готовности (не ждём всё сразу)
- **Параллельность:** несколько клиентов = несколько независимых потоков в разных вкладках

## Что маркетолог даёт другим блокам

| Блок | Что берёт из маркетолога |
|---|---|
| AI-Консультант | Боли ЦА → инструкции агента; сценарии поиска → быстрые кнопки |
| AI-HR | Аватары → описание идеального кандидата для найма |
| AI-Отдел продаж | Возражения + сегменты → сценарии тренировок |
| AI-Маркетолог (полный) | Сегменты → таргетинг рассылок; УТП → тексты постов |
| AI-Юрист | Ниша → типовые риски и договоры для этой сферы |
