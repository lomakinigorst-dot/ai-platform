## 1. СУТЬ И ПОЗИЦИОНИРОВАНИЕ

**Одной фразой:** Агентство подключает бизнесы — платформа сканирует сайт, анализирует ЦА и запускает набор AI-модулей, которые работают вместо отдела маркетинга, продаж, HR и финансов.

**Главное отличие от конкурентов:**
- Intly/b24u — только AI-консультант. Мы — 6 блоков из одной точки.
- WOW-момент при входе: добавил URL → через 5 минут рабочий виджет + готовый маркетинг-анализ + демо-ссылка — до оплаты.
- Маркетолог (ДНК-анализ) — фундамент для всех остальных блоков, запускается автоматически.

**Продаётся только через партнёров (интеграторов).** Прямая продажа онлайн — нет. Партнёр обеспечивает поддержку, установку, онбординг.

## 3. МОНЕТИЗАЦИЯ

### 3.1 Общий принцип
- Баланс как у хостинга (TimewWeb): общий счёт, с него списывается по каждому активному блоку.
- В сайдбаре: сумма на балансе + сколько дней до отключения при текущем расходе.
- Автопополнение с привязанной картой (порог настраивается).
- Каждый блок — отдельная подписка/пакет, видно в "Мои услуги".

### 3.2 Тарификация по блокам

| Блок | Единица | Логика |
|---|---|---|
| AI-Консультант (Лидоруб) | Пакет лидов | 1 лид = получен контакт. Пакеты: 50 / 150 / 500 / 1500 лидов. Неиспользованные сгорают через 30 дней |
| AI-Консультант (Поддержка) | Пакет диалогов | 1 диалог = любое общение с одним человеком в течение 24 часов. Пакеты: 500 / 2000 / 10 000 |
| AI-Маркетолог (базовый) | Включён | Автозапуск при добавлении клиента, базовый ДНК-анализ — бесплатно при любом блоке |
| AI-Маркетолог (полный) | Месяц | Рассылки, промо-акции, сквозная аналитика, колл-трекинг — отдельная подписка |
| AI-HR | Подписка/мес | За количество активных вакансий или за нанятых кандидатов (уточнить после анализа Eddler) |
| AI-Финансы | Подписка/мес | Тиры по количеству транзакций/банков |
| AI-Юрист | Пакет проверок | 1 проверка = 1 договор или 1 контрагент. Пакеты + подписка |
| AI-Отдел продаж | Подписка/мес | За количество менеджеров + безлимитные тренировки |
| AI-Оркестратор | Включён | Входит при наличии любого активного блока |

### 3.3 Триалы
- Каждый блок: 7–14 дней бесплатно
- Агент выдаёт триал из карточки клиента (кнопка)
- Триал маркетолога: за 7 дней клиент видит работающую рассылку → retention
- После триала — автоматическое предложение подключить с оплатой

### 3.4 Партнёрская подписка
- Агент платит фиксированную сумму за уровень (TBD)
- В уровень входит: N клиентов, N триалов в месяц, N сканирований
- Партнёр сам устанавливает цены клиентам (не ниже минимума платформы)

```
[Левая колонка — узкая, только иконки]   [Основное меню — разворачивается]

- Клик на иконку блока → разворачивается меню этого блока справа
- Активный блок подсвечен
- Заблокированные блоки (не куплены) — показаны серым с кнопкой «Попробовать 7 дней»

### 4.2 Верхняя панель
```

- Баланс всегда виден, клик → страница пополнения
- Уведомления: новый лид, ошибка канала, триал заканчивается

### 4.3 Режимы интерфейса
- **Простой** (по умолчанию): кнопки-сценарии, подсказки, минимум настроек

- Переключатель в боковом меню внизу

## 5. AI-ОРКЕСТРАТОР («ATLAS»)

**Концепция:** Главный AI-ассистент, который знает все блоки, может ими управлять и отвечать на вопросы о бизнесе клиента. Доступен из любого места через иконку в левом меню.

**Что умеет:**
- Отвечать на вопросы: «Сколько лидов сегодня?», «Кто из менеджеров косячит?», «Что по финансам за май?»
- Запускать процессы: «Создай вакансию на менеджера продаж» → идёт в HR-блок, запускает генерацию, возвращает на согласование
- Давать сводку по всем блокам: утренний брифинг при входе в кабинет
- Подсказывать: «Ваш триал маркетолога заканчивается через 3 дня, вот что уже сделано...»

**Интерфейс:** Чат-окно, раскрывается из иконки. Не отдельная страница — всегда под рукой.

## 8. БАЗА ЗНАНИЙ + AI-МАРКЕТОЛОГ (БАЗОВЫЙ)

### 8.1 Источники знаний
- Сканер сайта (рекурсивный, все страницы)
- Загрузка файлов: PDF, DOCX, TXT, CSV, XLSX
- Ввод текста вручную
- URL отдельных страниц
- URL сайтов конкурентов (отдельная функция — анализ и сравнение)

### 8.2 Структура базы знаний
**Разделы (авто + ручные):**
- Компания: описание, контакты, история
- Продукты/услуги: список, описания, цены, условия
- УТП и офферы ← синхронизируются с маркетологом (редактировать можно там И там)
- FAQ: часто задаваемые вопросы
- Скрипты продаж (из блока Отдел продаж)
- Возражения и ответы
- Конкуренты: сравнение, отстройка

**Логика синхронизации УТП:**
- УТП генерирует маркетолог → сохраняются в базу знаний автоматически
- Изменил в маркетологе → обновилось в базе → все AI-агенты видят актуально
- Удалил УТП — из базы тоже удаляется, но место не пустует — можно добавить вручную
- Все поля редактируемы в любом месте

### 8.3 Анализ конкурентов
- Добавить URL конкурентов через запятую → «Запустить анализ»
- DeepSeek сканирует их сайты + сравнивает с клиентом
- Выдаёт рекомендации: «У конкурента X лучше оффер — вот улучшенный вариант для вас»
- Клиент видит: **Было / Рекомендуется** — выбирает что применить (1, 2, 3...)
- Применённое → обновляется в базе знаний и у всех AI-агентов

### 8.4 Маскирование персональных данных
- При загрузке файла — автозамена телефонов, email, имён на [PHONE], [EMAIL], [NAME]
- Опцию можно отключить
- Защита от утечки PII через AI-ответы

### 8.5 ДНК-анализ — автозапуск (7 шагов)

**Запускается автоматически после сканирования сайта. DeepSeek обрабатывает блоками.**

| # | Шаг | Результат | Вкладка |
|---|---|---|---|
| 1 | Анализ ниши | Что за компания, продукт, регион, позиционирование | Ниша |
| 2 | Глубокий анализ ЦА | Боли, страхи, триггеры, возражения, мотивы (ДНК-промпт) | ЦА |
| 3 | Аватары клиентов | 3–5 детальных портретов покупателя | Аватары |
| 4 | Сценарии поиска | 6 блоков: сценарии, типология, семантика, негатив, сленг, цепочки | Поиск |
| 5 | Сегментация | 7 сегментов: портрет, каналы, тон, рекламные подходы | Сегменты |
| 6 | Заголовки и УТП | По каждому сегменту: 30 заголовков, 30 описаний РСЯ, 30 УТП | УТП |
| 7 | Автозаполнение агентов | Системный промпт, приветствие, быстрые кнопки, FAQ → все AI-блоки | — |

**Что получают другие блоки из маркетолога:**

| Блок | Что берёт |
|---|---|
| AI-Консультант | Боли → инструкции; сценарии поиска → быстрые кнопки; возражения → скрипт |
| AI-HR | Аватары → идеальный кандидат; ниша → текст вакансии |
| AI-Отдел продаж | Возражения + сегменты → сценарии тренировок |
| AI-Маркетолог (полный) | Сегменты → рассылки; УТП → тексты постов |
| AI-Юрист | Ниша → типовые риски отрасли |
| AI-Финансы | Ниша → типовые статьи расходов отрасли |

## 9. AI-МАРКЕТОЛОГ (ПОЛНЫЙ БЛОК)

### 9.1 Промо-акции и рассылки
- 1 клик → генерация промо-акций на месяц/квартал/год на основе базы знаний
- Автораскладка в календарь
- AI знает ЦА → автосегментация: кому сейчас, кому через 30 дней
- Сегменты рассылок: новые лиды / купившие 30+ дней назад / постоянные / отказники / по этапу сделки

### 9.2 Генерация контента
- Посты для соцсетей: выбрать сегмент + стиль + длина → текст + промпт для изображения
- Планировщик публикаций: черновики, расписание, история

### 9.3 Сквозная аналитика (Фаза 2)
- Яндекс.Директ: расходы, кампании, ключи
- Яндекс.Метрика: поведение, конверсии
- CRM: источник → лид → сделка → деньги
- Колл-трекинг: подменные номера по источникам (отдельный тарифный пакет)
- Авито как источник — нативно

### 9.4 Бесплатное vs Платное
**Бесплатно (с любым блоком):** ДНК-анализ (шаги 1–7), базовые УТП, автозаполнение агентов
**Платно (полный блок):** рассылки, промо-акции, календарь, генерация постов, сквозная аналитика

## 10. AI-КОНСУЛЬТАНТ (БЛОК 1)

### 10.1 Две роли
**Лидоруб (Менеджер продаж):**
- Консультирует → выясняет потребность → берёт контакт
- Цену не называет — передаёт менеджеру
- Тариф: пакеты лидов. 1 лид = получен контакт.
- Резюме диалога: суть, приоритет, контакты, UTM

**Поддержка (Специалист):**
- Знает всё по базе, честно говорит «не знаю» если нет в базе
- Тариф: пакеты диалогов. 1 диалог = общение 24 часа с одним человеком.

### 10.2 Виджет
**Внешний вид:**
- Плавающая кнопка (правый нижний угол)
- Реалистичные фото (мужской/женский, деловой стиль)
- Список имён на выбор или своё фото (автообрезка в круг)
- Цвета, размер, позиция — настраиваются

**Триггеры появления:**
- Таймер (N секунд)
- Exit intent (мышь к верхнему краю)
- Scroll depth (X% страницы)
- Бездействие (N секунд)
- Повторный визит

**Контекст страницы (автоматически):**
- URL текущей страницы, заголовок
- UTM-метки, источник трафика
- JSON-LD (цены, наличие, рейтинги) — если есть разметка на сайте
- Поведенческие данные для аналитики

**Аналитические события (для GA/GTM/Яндекс.Метрики):**
- Виджет открыт / закрыт / свёрнут
- Контакт получен (email/телефон)
- Лид передан в CRM

### 10.3 Настройки AI
- Порог эскалации: ползунок 0–100% с визуализацией зон
- Имитация печати: скорость (симв/сек), вариативность ±%, макс задержка
- Температура, макс токены, длина ответа
- Системный промпт: автозаполнен после ДНК-анализа, редактируется в режиме Эксперт
- Версионирование промптов

### 10.4 Каналы
Telegram, WhatsApp, ВКонтакте (MAX), Авито, виджет сайта

### 10.5 Интеграции и передача лидов
- Email: резюме диалога + контакты + UTM
- Telegram-бот агента: уведомление в реальном времени
- Битрикс24 / AmoCRM: создание лида/контакта
- После подключения CRM → AI видит историю клиента из CRM

### 10.6 Онбординг нового клиента — предложение подключить CRM
После первой настройки: «Куда передавать лидов? Подключите CRM одним кликом» → клиент нажимает → авторизация → готово. WOW: «Теперь лиды из виджета сразу попадают в [Битрикс24]»

### 10.7 Воронка лидов (Канбан)
Столбцы: Новый → Первый контакт → Квалифицирован → КП → Переговоры → Закрыт
Лиды из виджета — автоматически в «Новые»
Автономия: переключить включить — AI сам двигает лидов по воронке

### 10.8 Аналитика
- Диалогов / лидов / конверсия / уверенность AI / эскалации
- Тональность диалогов
- Топ-5 вопросов посетителей
- «Сэкономлено часов поддержки»

### 10.9 Демо-ссылка (WOW-момент)
**Механика:**
- После сканирования и настройки → URL вида: `platform.ru/demo/имя-домена-клиента`
- Копируется HTML главной страницы клиента (только скелет, минимальный вес)
- Добавляется наш виджет
- Агент отправляет клиенту → клиент открывает → видит свой сайт + работающий виджет

## 11. AI-HR (БЛОК 3)

### 11.1 Конкурентное преимущество над Eddler
Eddler теряет 50% кандидатов — ссылка только через HH.
Мы отправляем через все каналы: HH + WhatsApp + Telegram + MAX + SMS → доставляемость ~100%.

**Eddler:** ~350 млн ₽/год. Рынок проверен.

### 11.2 Воронка найма
1. AI генерирует вакансию из базы знаний компании + должности
2. Публикация на HH через API
3. Кандидат откликается → автоответ во все каналы (приоритет: HH → почта → Telegram → WA → MAX → SMS)
4. Сообщение: «Привет! [Компания] получили ваш отклик. Пройдите знакомство → [ссылка]»
5. Кандидат проходит демо-лендинг (5 страниц)
6. Результаты → в кабинете HR

### 11.3 Демо-лендинг кандидата (5 страниц)
1. Приветствие и кто мы
2. Что продаём/делаем
3. Почему стоит работать у нас
4. Тестовые задания и вопросы
5. Следующий шаг (звонок / встреча)

### 11.4 Обучающая платформа
- Программы обучения новых сотрудников
- Тесты и задания
- Отслеживание прогресса

> ⚠️ Полный функционал — после анализа Eddler (Игорь предоставит доступ)

## 15. ДИЗАЙН

### 15.1 Визуальный стиль (из референсов)
- **Фон:** белый или очень светлый (светло-серый #F8F9FA или светло-фиолетовый #F5F3FF)
- **Акцент:** один цвет — фиолетовый (#7C3AED) или синий (#2563EB)
- **Карточки:** белые, тонкая тень, скруглённые углы (border-radius 12px)
- **Метрики:** крупные цифры + стрелка изменения (↑4.9% зелёный / ↓2.1% красный)
- **Шрифт:** Inter (Google Fonts, везде)
- **Таблицы:** минималистичные, без лишних линий
- **AI-ассистент:** панель справа (как AudiencePlus Copilot) — выдвигается по кнопке

### 15.2 Референсы (папка «Нравятся дизайны»)
- FinboTech — метрики + таблицы, светло-фиолетовый фон
- AudiencePlus — правая AI-панель, тёмный сайдбар + белый контент
- Finexy — карточки с дельтами, оранжевый акцент, продажи-ориентированный дашборд
- Windpo — серый фон, минималистично, список слева + детали справа

## 16. ТЕХНИЧЕСКИЙ СТЕК

| Слой | Технология |
|---|---|
| Backend | Python 3.12 + FastAPI |
| БД | PostgreSQL + pgvector |
| Очереди | Celery + Redis |
| Frontend | Next.js 14 + TypeScript + Tailwind CSS |
| Виджет | Vanilla JS (без зависимостей) |
| AI основной | DeepSeek API (прямой, не через агрегаторы!) |
| AI аналитика/тексты | Claude Sonnet (Anthropic API) |
| Хранилище файлов | Yandex Object Storage |
| Инфра | Docker + Nginx |
| Краулер | Python (aiohttp + BeautifulSoup) на сервере |

> ⚠️ Краулер работает на сервере, не в браузере. Прогресс отдаётся через WebSocket — пользователь видит живые обновления, но вкладку не нужно держать открытой.
