# Список фич от конкурентов — на согласование

> Формат: **Название** — что это, зачем, в каком блоке работает.
> Оставь то что хочешь — остальное удали.

---

## ЯДРО / КАБИНЕТ ИНТЕГРАТОРА

### 1. Переключение «Агент / Мой бизнес» в одном аккаунте
Интегратор может одним кликом переключиться из режима «управляю клиентами» в режим «смотрю на свой бизнес как клиент». Не нужно два разных аккаунта.
**Блок:** Ядро (все блоки)

### 2. Здоровье каналов — статус одной строкой
В дашборде агента: «3 из 5 каналов работают нормально» с цветовым индикатором. Сразу виден сбой без входа в каждого клиента.
**Блок:** Ядро / AI-Консультант

### 3. Команда с ролями и правами
Пригласить сотрудников в агентство: менеджер по продажам, аккаунт-менеджер, технарь. Каждому — своя зона доступа (финансы, клиенты, настройки AI).
**Блок:** Ядро

### 4. Экспорт CSV везде
Кнопка «Экспорт CSV» в списке клиентов, транзакциях, сканах, лидах. Нужна для передачи данных в Excel, CRM, аналитику.
**Блок:** Ядро (все блоки)

### 5. Реферальная программа
Партнёр приглашает другого партнёра — получает % от его оборота. Cookie 90 дней. Простая страница с реферальной ссылкой и статистикой.
**Блок:** Ядро

### 6. Финансы агента — история транзакций + hold
Доход по дням (график), hold-период перед выплатой, кнопка «Запросить выплату», реквизиты. Всё в одном разделе.
**Блок:** Ядро

## AI-МАРКЕТОЛОГ (базовый блок, работает для всех)

### 7. Автозапуск маркетинг-анализа сразу после сканирования
После того как сканер обходит сайт — автоматически запускается ДНК-промпт (7 шагов: анализ ниши, ЦА, аватары, сценарии поиска, сегментация, РСЯ-заголовки). DeepSeek обрабатывает по блокам. Интегратор не делает это руками — всё происходит само.
**Блок:** AI-Маркетолог (триггерит после сканирования)

### 8. Вкладки результатов маркетинг-анализа
Результаты ДНК-анализа разбиты по вкладкам внутри карточки клиента: «Ниша», «ЦА», «Аватары», «Сценарии поиска», «Сегменты», «Заголовки/УТП». Можно открыть любой раздел отдельно.
**Блок:** AI-Маркетолог

### 9. Генерация постов для соцсетей
Кнопка «Создать пост»: выбрать сегмент, стиль (профессиональный/дружелюбный), длину → AI генерирует текст + промпт для изображения. Каналы публикации подключаются через интеграции.
**Блок:** AI-Маркетолог

### 10. Планировщик публикаций
Черновики, запланированные посты, история публикаций. Выбрать дату/время → опубликовать.
**Блок:** AI-Маркетолог

## AI-КОНСУЛЬТАНТ (виджет)

### 11. Имитация печати человека
Виджет «печатает» с задержкой как живой человек. Настройки: скорость (симв/сек), вариативность ±%, макс. задержка. Повышает конверсию — клиент чувствует живое общение.
**Блок:** AI-Консультант

### 12. Порог эскалации с визуализацией
Три зоны: 0–50% уверенности AI → эскалация на оператора, 50–70% → уточняющий вопрос, 70–100% → автоответ. Наглядный ползунок в настройках.
**Блок:** AI-Консультант

### 13. Контекст страницы в реальном времени
Виджет автоматически считывает URL страницы, UTM-метки, источник трафика, JSON-LD (цены, наличие, рейтинги). AI отвечает с учётом того, что клиент смотрит прямо сейчас.
**Блок:** AI-Консультант

### 14. Аналитические события для GA/GTM/Яндекс.Метрики
Виджет отправляет события: открыт чат, закрыт, получен контакт (email/телефон). Клиент видит в своей аналитике сколько лидов принёс виджет.
**Блок:** AI-Консультант

### 15. Канбан-воронка лидов
Столбцы: Новый → Первый контакт → Квалифицирован → КП → Переговоры → Закрыт. Лиды из виджета падают автоматически в «Новые». Дополнительные столбцы — рассылки, события, источники, формы.
**Блок:** AI-Консультант / AI-Отдел продаж

### 16. «Уверенность AI» в аналитике
Метрика по каждому клиенту: средний % уверенности AI в ответах за период. Если падает — значит база знаний устарела или агент не обучен на новый контент.
**Блок:** AI-Консультант

### 17. Тональность диалогов
Диаграмма по тональности входящих сообщений: позитив / нейтраль / негатив. Позволяет поймать волну недовольства клиентов до эскалации.
**Блок:** AI-Консультант

### 18. Ответы на отзывы (WB / Ozon / Яндекс.Маркет)
AI автоматически отвечает на отзывы на маркетплейсах. Отдельный тариф за отзыв. Актуально для клиентов с e-commerce.
**Блок:** AI-Консультант

### 19. Маскирование персональных данных в базе знаний
При загрузке файлов в базу знаний — автозамена телефонов, email, имён на [PHONE], [EMAIL], [NAME]. Защищает от утечки PII через AI-ответы.
**Блок:** AI-Консультант / База знаний

### 20. Доступ агентов к базе знаний
При загрузке файла в базу знаний — выбрать каким AI-агентам он доступен (консультант видит одно, HR-агент — другое).
**Блок:** Ядро / База знаний

## AI-HR

### 21. Воронка кандидатов (Канбан)
Аналог лидов, но для найма: Отклик → Скрининг → Собеседование → Оффер → Принят/Отказ. Кандидаты из HH падают автоматически.
**Блок:** AI-HR

### 22. Шаблоны ответов кандидатам
Библиотека сообщений для разных этапов воронки: приглашение на собеседование, отказ, оффер. AI адаптирует шаблон под конкретного кандидата.
**Блок:** AI-HR

## AI-ОТДЕЛ ПРОДАЖ

### 23. Анализ тональности разговора
После загрузки записи звонка — AI выдаёт: тональность менеджера, тональность клиента, моменты напряжения, пропущенные возражения.
**Блок:** AI-Отдел продаж

## ТАРИФЫ И МОНЕТИЗАЦИЯ

### 24. Тарификация по потреблению (за диалог)
Пополнение баланса клиентом → списание за каждый диалог (простой / сложный / отзыв / рассылка). Клиент понимает за что платит. Партнёр ставит свою наценку.
**Блок:** Ядро / Монетизация

### 25. Таблица сравнения тарифов
Большая визуальная таблица: строки = функции, столбцы = тарифы, ячейки = ✓ / ✗. Клиент сам выбирает нужный уровень. Конвертирует лучше текста.
**Блок:** Ядро / Лендинг

### 26. Баланс всегда виден в сайдбаре
Текущий баланс клиента в левой панели с кнопкой «Пополнить». Клиент никогда не забывает что деньги заканчиваются.
**Блок:** Ядро / Клиентский кабинет

## ОНБОРДИНГ

### 27. Чеклист онбординга нового клиента (для агента)
Пошаговое руководство внутри кабинета агента: добавить сайт → проверить чат → задать вопросы клиенту → демо → передать виджет. Агент не забывает ни один шаг.
**Блок:** Ядро / Онбординг

### 28. Демо-ссылка сразу после сканирования
После анализа сайта — автогенерируется демо-страница с виджетом клиента. Агент отправляет её клиенту: «Вот уже работает» — WOW-момент до подписки.
**Блок:** Ядро / WOW-момент (уже в ТЗ, подтверждается конкурентом)
